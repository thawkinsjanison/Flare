Payment Settings - Introduction

If you require the facility to apply payments to the learning you deliver then this is possible using the CLS Payment feature. Payments can be applied to one or more Catalogue Items once the settings have been configured in the Payment Settings area of the CLS.

Payment Settings can be accessed by navigating to Settings > Payment Settings.

Note: In the CLS credit card numbers are sent encrypted to the payment provider. The Janison CLS only stores the receipt number from the payment provider and the last 4 digits of the credit card number.

Merchant accounts

A merchant account is a type of bank account that allows businesses to accept payments by payment cards, typically debit or credit cards.

Payment gateways

A payment gateway is a server that is dedicated to linking websites and banks so that online credit card transactions can be completed. In this case, the payment gateway facilitates the transfer of information between the Janison CLS, and the Front End Processor or acquiring bank.

Paypal

The Janison CLS supports PayPal as the third party processor. For more information about PayPal account types please see the Paypal website.

Changing Currency

You can change the currency by clicking the Languages link.

This takes you to Settings > Languages

Currency Culture – select the country currency you wish to use.

Currency Prefix - refers to the letters prefixing the quantity of money. There are four options available. The option (ISO Currency Code), refers to the International Organisation for Standards set codes for currency. Activate the drop-down menu and select the required code.

Format – Displays an example of how the currency will look once you have selected the top two options.

Default Culture – the code chosen will only be used when no culture in the User’s browser settings match supported languages or multilingual Support is off (optional). Please note that setting this to None will inherit the Global tenant value.

Date Format Culture – it is here that you can determine the formatting of the date which will be used throughout the site. For example English (Australia) for {dd,mm,yyyy} or English (United States) for {mm,dd,yyyy}.

Multilingual Support – if checked allows for two languages to be available for selection.

Currently Active – displays the languages that have been selected to be active on the site.
