Resource Options

The following options are available within this section.

Show Culture Code

This option enables the User to set a language other than English. It is possible to set a number of languages and switch between them.

Enable Workflows for Resources

A workflow is a review system that allows the appointment of multiple reviewers during question development in order to manage the quality and approval process. Workflows can be attached to Questions, Resources and Stimulus. If checked a workflow status will display on the Resource details page only if the Resource is linked to an Assessment Module where a workflow status has been set. The workflow status can be changed by clicking on the widget highlighted in the screenshot below. Please be aware that transitions will only appear if the User has the correct permissions assigned through their role.

Enable Comments for Resources

If checked a comments section will appear on the details page of the Resource.  A comment or question can be posted and is then available for others to view, like or dislike and finally to reply to. The comment can be edited by clicking on the pencil icon. Additionally, if Post Visible is unchecked the post will be unavailable for others to view.

Enable resource custom attribute set on discipline

If checked the ability to link a custom attribute to a Resource is available. It is useful to filter, sort, search and report. For more information please see the document on Custom attributes.

Enable versioning for resources

Versioning is the assignment of unique version numbers to unique states of software. Versions are made up of major and minor components. The major version is to the left of the decimal place, the minor to the right. For example 1.53 has a major version of 1 and a minor version of 53.

If checked a Version field with a version number will appear on the details page of the Resource. The Version number will help to keep track of any changes made to the Resource.

For further information please see the document on Versioning.

Enable Resource Name Field

This setting will includes a mandatory field where the Resource needs to be given a Name as well as an Identifier. Population of the Name field will be mandatory at Resource creation and will appear on the edit form as seen below.
