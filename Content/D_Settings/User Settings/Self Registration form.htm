Self-Registration form

The details outlined here will determine what is seen on the self-Registration form to be filled out by new Users if they are registering themselves.

Navigate to Settings > User Settings > Self-Registration Form.

  1. Self-Registration - these can either be Disabled or enabled by selecting Registration form on a separate page. If enabled a link to Register will appear on the login page.

  1. Skip email validation on self-registration -check this box if you don’t require the User to validate their registration through their email account. If left unchecked the User will receive a Registration Confirmation email with a link to activate their account and to set a password.
  2. Auto-generate password on registration - check this box if you require the system to generated passwords. This option is only available if Password Generation has been enabled on the Password Settings Page.
  3. Allowed Email Masks – restricts registration to users with the specified email address. In the example below it has been configured to only allow email addresses ending in .com.au to register successfully.

  1. Username generation – activate the drop-down menu to select the format the system will generate the usernames in.

  1. Main Role – select a main role to be assigned to self-registering Users. This will usually be the basic user Role.

  • Visible – check the boxes beside the fields you would like to appear on your registration form.
  • Mandatory – checking this box makes the field mandatory. This will be recognisable by a red asterisk beside the field on the form.
  • Capitalisable – if this box is checked any data entered into the field will automatically be capitalised.
  • Order – drag the arrows to change the order the fields will appear on the form.

Validation check – on submission of the self-registration form, checks that the populated field is unique in the system. In the screenshot below Email has been set for the Validation check, the User who is registering must enter an email that does not exist in the system. If the email exists in the system the User will encounter a warning notifying them so.

Using Groups to auto-enrol users into Catalogues

Group Type – setting a Group Type on the Self-Registration form allows for powerful functionality to be included enabling Users to be automatically enrolled into different Catalogue Items depending on their grouping. In the example we have used below you can enrol children in relevant courses depending on their age range.

There are a number of steps involved in this process.

Step 1 – Involves setting a Group Type (this will be the text you would like to appear on the Self-Registration form).

  • Navigate to Manage People > Group Types > Add Group Type.
  • Create the new Group Type Name and add Groups before selecting Save Group Type.

Step 2 – Involves ensuring the Group Type appears on the Self-Registration form.

  • Navigate to Settings > User Settings > Self Registration Form.
  • Activate the Group Type drop-down menu and select the newly created Group Type. For the purpose of this example we have created a Group Type titled ‘Your age range’.
  • Click Save Settings. You will see the New Group Type appear in the field list above and will be checked automatically.

Step 3 – Involves setting up User Enrolment Rules which links the Group Type to an automatic enrolment for a Catalogue Item.

  • Navigate to Settings > User Enrolment Rules > Add User Enrolment Rule.
  • Enter a Name for the User Enrolment Rule.
  • Activate the drop-down menu What does this Rule enrol users for and select a Catalogue Item. This may be a course that you want certain Users automatically enrolled in when they self-register.

  • Add a filter to point to the Group you are referring to by using the (Any Group) field. In the example provided the value is the age range 10-12. This means that any Users that select the Group Type 10-12 years will automatically be enrolled in the Amazing Facts Course upon self-registration. See instructions about how to set these up at Building Filters.
  • When should the rule apply, check On Self Registration.
  • Click Save User Enrolment Rules.

On the Logon page click Register and check the Self-Registration form for the newly added information.
