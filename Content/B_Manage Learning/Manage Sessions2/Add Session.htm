Add Session

You cannot create a Session in isolation because it must belong to a Learning Event:

You add a Session to an existing Learning Event and you can reach this functionality via several different screens, for example when you are inspecting the Learning Event's details screen or editing it.

In Manage Sessions you can also select Manage Learning > Manage Sessions and click Add Session.

You are then taken to the Manage Learning Assets section of the CLS to the New Learning Event Session screen where you can add a Session to an existing Learning Event.

  1. Click and select a Learning Event for this Session.
  2. Select a Status
  • Setup - not published to the Catalogue.
  • Published - published into the Catalogue.
  • Archived.
  • Completed – all attendees have been marked and the Session is completed.
  • Cancelled.
  1. Click and select the appropriate Time Zone.
  2. Use the date and time pickers to set Start and End dates and times.
  3. Use the date and time pickers to set Registration Open and Close dates and times.

  1. Select Hours or Days as the duration unit.
  2. Enter the duration using these units.
  3. Set the minimum and maximum attendance.

    4. You will be warned later if you enter a value for the maximum attendance that is larger than the capacity of the Venue you choose at 9.

    Note that if you leave the Max Attendance field blank, the capacity of the Venue that you select in 9 will populate this field. You can override this.

  1. Click and select a Venue, or
  2. Click QuickAdd Venue to create a new Venue and enter its name.

    3. Enter its capacity, select a Location and click Save.

    You can now select this new Venue at 9.

    You can also create a Location for this Venue at the same screen if you click Create location now before you save.

    A new Location will be created with the same name as the Venue and you can enter other location details. The address is important if you want learners and administrators to see Google maps of the Location.

    If you have previously selected an existing Location for this new Venue, the button changes and you can click Edit location now if required.

    When you click Save, the Venue and the Location will both be saved.

    Note that when you add or edit a Location here, the same button changes again and you can cancel the Location create/edit with this button if needed before you click Save. Cancelling the creation of a Location will remove the name of the new Location from the Venue's Location box and allow you to select another Location from the drop-down list before saving the new Venue. Cancelling Location edit will leave the Location unchanged.

  3. Click the checkbox to show learners a map for the Location of this Session. Remember that an address needs to have been entered for the Location so that Google maps can find it.
  1. To include attachments for learners use the Upload Files feature.
    Click Select Files one or more times to include attachments. You can remove a file using the little cross icon on the right hand side.

    2. Note that because of the danger of overwhelming an email system, these attachments are not sent directly to attendees. Instead, they receive a link in their email/iCAL appointment that allows them to view the attachments via the Session Summary screen in the CLS.

  1. Click and select a name from the drop-down list then click the plus icon to add facilitator(s) for the Session. You can do this multiple times. You can remove a name using the minus icon on its right hand side.
  2. You can provide extra information about particular Sessions using the rich text editor. Users who are not attending will not see these instructions.

    3. To include an email address as a link, select Insert > insert link. Enter mailto:admin@email.com for the Url address, replacing this email address with the required address.

    The information in Instructions will be provided to attendees in two ways:

  • In the CLS, on the Session Summary screen.
  • In the iCAL email that is sent by the CLS to all attendees when they are enrolled in a Session.
