Certificate Number generation

Certificates can have a unique number if the field CertificateNumber is included in the template. The number is generated by the CLS.

By default it is specified technically as: {code}{########}/{year}.

This means that it is constructed from 3 sources:

  • [code} =the first four characters of the Learning Asset identifier
  • 8 numbers from the unique User enrolment number (technically, the progress guid)
  • /the current year

You can overrule this through Tenant Settings. In the example below the Certificate number has been set to ACPCert{#####}.

Since the Certificate number has been set to ACPCert{#####}, the Certificate number will be constructed from:

  • {code} = ACPCert
  • 5 numbers from the unique User enrolment number

There is no reference to the year.

The Certificate Code can also be changed for an individual Learning Asset during creation or editing of the Learning Asset or Catalogue Item. For example, enter 00001 in the Certificate Code field to add 00001 to the start of all its Certificate numbers.

In this example the Certificate code will be constructed:

  • {code} = Certificate Code, i.e 00001
  • the 8 numbers from the User’s enrolment, e.g  21250869
  • /the current year
