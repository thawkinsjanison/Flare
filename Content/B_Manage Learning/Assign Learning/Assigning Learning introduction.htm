Assigning Learning - introduction

For Users to participate in the learning and assessment available in the CLS they need to be enrolled in Catalogue Item(s) and hence, assigned learning. This involves:

  • The learning content, i.e. the Catalogue Item(s)
  • The recipients, i.e. the Users to be enrolled
  • The delivery options, i.e. enrol now or in the future

There are a number of different ways that enrolments can be done, see Enrolments for more information. For example, the process can be done manually by assigning learning directly to selected Users from within a particular Catalogue Item, see Assign Learning – Catalogue Item. It can also be done by uploading a spreadsheet of Enrolment data containing lists of Catalogue Items, Users, Status, and so on, see Import Learning History. Alternatively you can use Assign Learning, a wizard that creates Enrolments semi-automatically. Using the wizard you are presented with a series of screens and you fill them in to nominate when and how learning will be delivered and to whom.

Once User Enrolments have been added to the CLS, they can be inspected and edited in the Enrolments section of the CLS.

Note that a Catalogue Item that contains more than one Learning Asset becomes known as a Program but remains functionally the same (see Programs). Thus, references to Catalogue Items here also include Programs.
