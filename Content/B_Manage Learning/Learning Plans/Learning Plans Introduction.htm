Learning Plans Introduction

A Learning Plan is a plan in which an individual schedules particular learning that they would like to do, following an analysis of what knowledge and skills they need to achieve their career and personal goals. Learning Plans are often established for a particular period of time, for example one year, and are often subject to a management workflow – i.e. a manager needs to approve the learning plan submitted by a staff member. These functions and more are supported by learning plans in the JanisonCLS.

Learning Plans are a kind of learning asset in the JanisonCLS. They function in almost all respects like any other learning asset. They can:

  • Be added to catalogue items
  • Be enrolled in
  • Be completed
  • Be subject to enrolment rules and group enrolments

Learners can also have more than one learning plan at once, provided the learning asset is added to more than one catalogue item – you cannot have more than one enrolment in a catalogue item at once.

Learning Plan

At the highest level is the learning plan itself. The learning plan is personal to the learner and contains

  • A statement about the learner’s broad goals;
  • A list of the learner’s managers and supervisors;
  • A number of specific learning objectives (referred to as Learning Plan Entries); and
  • A journal.

Learning Plan Entries

The learner enters their broad goals and direction at the learning plan level. After this, the learner will determine a number of specific objectives that they would like to accomplish. These are recorded as Learning Plan Entries.

Each entry has a number of attributes

  • Description of the learning objective;
  • What action the learner will take to meet that objective;
  • A date by which the learner would like to meet that objective;
  • The date on which the learner actually did meet that objective;
  • A number of catalogue items that the learner will or has completed to help meet this objective; and
  • Indicators of whether the objective is active and/or complete.

 

Learning Plan Entries can be added or removed at will by the learner unless they have been made mandatory by an administrator (see below).

The journal.

The journal serves three functions.

  1. It is a way for the learner and their manager to record comments as part of the approval process;
  2. A place for the learner to reflect on the learning they have undertaken and how effective it has been; and
  3. A place for the learner to upload documents that are valuable artefacts for the learner to use in the future for job applications, professional development audits etc. These might include (but are certainly not limited to) a CV, letters of reference, certificates and qualifications.

Recording accomplishments

When a learner completes an item from the catalogue in the JanisonCLS, that accomplishment can be linked back the learning plan.

If the learner undertakes learning or professional development outside the JanisonCLS, they can get credit for that too using the “Log external learning” feature.

Activating the plug-ins

Your JanisonCLS will not support Learning Plans if you don’t have the “Plans” plug-in activated.

If you are using the “Log external learning” feature, you will need to have the “Other Learning” plug-in activated.

If you don’t have these activated, you will need to contact the Client Services Team at Janison. It will only take a moment to activate it.

Doing the background work

Before your users can have a learning plan, you need to do some set up work.

Firstly, ask yourself some questions.

Do I want my users to link their learning plans to a particular set of professional standards or something similar?

If so, you will need to set up a tag for each standard and group them into one or more tag types. If you need help doing that..

Do I want to group my learning in any way? Some organisations want learning grouped into learning areas or need to record which competency certain pieces of learning relate to. This is particularly relevant if your learners need to meet certain CPD or similar requirements.

If so, you need to set up a tag for each learning area and group them into one tag type.

Do I want my users to be able to record learning they have done outside the JanisonCLS against their learning plans?

If so, you will need to set up a learning asset of type “Other Learning” and link it to a catalogue item. In addition to the basic steps for completing this you will need to do four things:

  1. Check the “Allow learner to link to learning plan entry” box when you are editing the Learning Asset

  1. Give the Catalogue Item a name that will mean something to your learners. This is particularly important if you are going to use more than one catalogue item so that external learning can be recorded against different groups (see advanced user instructions below). Your learners will see the catalogue items in a drop down menu when they are editing their learning plans

  1. If your learners need to record a course accreditation number when they record their learning, you need to check the “Allows accreditation number” box

Be careful! If you are using this, it’s probably because some of the learning your learners will do outside the JanisonCLS will be accredited and some won’t be. Do not put a Course Accreditation Number on the catalogue item as well. If you do and the learner doesn’t enter a Course Accreditation Number when they record their learning, it will default to the one you put on the catalogue item.

  1. Make the catalogue item private (i.e. don’t publish it in the catalogue). If you need help with doing this.
