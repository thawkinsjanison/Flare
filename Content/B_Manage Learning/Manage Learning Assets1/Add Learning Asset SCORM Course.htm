Add Learning Asset - SCORM Course

SCORM stands for Shareable Content Object Reference Model. SCORM courses are courses that have been created in 3rd party applications such as Lectora and Captivate. They can be exported to disk (as SCORM compliant zip files) and imported into the CLS.

Learner Users enrol in a Catalogue Item that contains the SCORM course Learning Asset. This launches the SCORM course player in a separate browser window. User progress through the course is tracked by the CLS and classified as Not Attempted, Incomplete or Complete as appropriate.

Before adding this type of learning make sure that your file is a SCORM compliant zip file. Note that the default upload limit is 200MB. If you require a larger limit please contact the Janison Development team with a request to increase the size.

To add a SCORM Course, browse to Manage Learning > Manage Learning Assets.

  1. Select Add Learning Asset.

  1. Select New SCORM Course.

  1. Select Select files to browse to the SCORM zip file on your computer and select Upload.

A progress bar will inform you when the upload is complete.

  1. Enter the Name and Identifier

  1. Optionally, enter Information for your records

  1. Optionally, select the Competencies section.
  2. Select a desired Competency.
  3. Select the icon to add it.

You can add multiple competencies as needed.

  1. Optionally, Select the Advanced Settings section.
  2. Select an Internet Explorer Compatibility Mode.

Some older SCORM courses do not play in later versions of Internet Explorer. This setting allows the course to be forced into a specific IE mode to ensure learners using all EI versions will be able to play the course.

  1. Select Save SCORM Course .

Preview SCORM

Once you Save SCORM Course you are taken to that SCORM Course’s details page.

  1. Select Preview.

Preview will run the SCORM course in a preview mode. No progress will be tracked and the JaniosnCLS will not retain any scores or completion statuses. If the SCORM preview is refreshed or the tab is closed you will have to start again from the beginning of the SCORM.

Setting up Roles for preview.

As well as having the SCORM Player plugin enabled, you will need to enable certain settings within the roles menu for this feature to work correctly.

Browse to Settings > Roles.

  1. Select Basic User.

  1. Select the icon in the Securable Areas of the application section.

  1. Select Allow for Scorm Player > Player > Course Player Preview.

  1. Select Save.

You have now enabled the ability for all Basic Users to see the SCORM player in a course panel.

Return to Settings > Roles.

  1. Select Tenant Admin.

Alternately select the role which is used by your content curators.

  1. Select the icon in the Securable Areas of the application section.

  1. Select Allow for Scorm Player > Player > Preview.

  1. Select Save.

You have now enabled users in the Admin role to preview SCORM courses in a course panel.
