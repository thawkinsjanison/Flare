New Features

Subscriptions

Feature Documentation

Subscriptions is a new way to apply discounts on a suite of content. Using subscriptions, you are able to easily setup and manage discounted bundles of learning. Learners are able to purchase subscriptions which then give them discounted access to learning. Subscriptions are highly customizable and can be configured in a multitude of ways.

New and Improved UI for Program Details view

Learners now have a much cleaner and more intuitive UI for enrolling and launching programs with multiple components. The new program template clearly displays your progress and status in each component of the program as well as extra details on each type of asset within the program. Contextual information is also provided based on the asset type of each component, for example, if a program contains a Learning Event you will be able to manage your session right from the program interface before enrolling.

The new program template is a 1-size-fits-all template and will be replacing the Default Vertical template as the default program style. As such, Janison will no longer be supporting the Default Vertical template as new features and adjustments are made to the CLS.

To take advantage of the new program UI, clear the Template selection in the Edit Catalogue Item settings menu. This enables the new default template.

Conditional Release of Program Components

With the new Conditional Release feature, you are able to set the access of a program component to allow or block access based on time and date rules. Using this feature you can now develop and customise in-depth programs which restrict access to certain components until a set date or time after enrolment. The same rules can also be applied to block a component after release.

New Left-hand Navigation Option for the Janison Course Player

Course Modules can now support left-hand navigation with the addition of the Nested Vertical Navigation Type. As well as providing a new look and feel to course modules, Nested Vertical also supports nested module pages. This makes it easier to read and navigate between nested pages within Course Modules.

To set left-hand navigation use the Navigation Type selection within a Module’s settings.

New Authentication Mechanisms

Added support for the following Single Sign-On (SSO) mechanisms.

  • Facebook
  • Google
  • Office 365 (Azure AD)

Improvements and Enhancements

CPD Plan Enhancements

CPD Plans have received some enhancements which increase their functionality and provide Learners with more clarity on their progress.

  • Added support for Fractal Points which allow for point/hour requirements with decimals as opposed to only whole numbers.
  • Added Total Points to Learners CPD dashboard tile to provide a complete picture of CPD plan status.
  • Improved Learner CPD dashboard tiles when displaying CPD plans with many tag category requirements.

CPD Reporting

Off the back of the CPD Plan Enhancements, we also added two new Customisable Reports for CPD to help managers better track and consolidate data on their team’s CPD memberships.

  • Added a Customisable Report CPD Details Report which shows a breakdown of CPD by tags.
  • Added a Customisable Report CPD Summary Report which shows a breakdown of CPD Memberships for each user.

Included in the new CPD reports is an abundance of CPD reporting APIs which you can filter by in your Customisable Reports.

Discussion Forum Enhancements

Feature Documentation

Discussion forums can be configured as Announcements. Announcements act in the same way as forums do, however, only the discussion forum administrator is allowed to make posts on the Announcement.

Discussion Forum Settings were added to the Discussion Forum itself, allowing you to set different settings per forum.

Learning Plan Enhancements for Managers

Added the ability for Managers to View and Comment on a learner’s Learning Plan.

New Catalogue Item Access Restriction Rules

Expanded catalogue item access restriction rules by adding the ability to:

  • Restrict access to catalogue items after a period of time based on the due date.
  • Restrict access to catalogue items after a period of time based on the enrolment creation date.

Subdomain Application URL Enhancement

Where originally Application URL could only be set to the Tenant level, within 1606 Application URL has been enhanced to accommodate Clients who use custom URLs for multiple Subtenants allowing you to set a unique URL for each Subtenant available to you.

These settings can be found under Settings > Email Settings.

User Enrolment Rule Enhancements

We have made a number of improvements to User Enrolment Rules to increase performance and logging capabilities as well as expanding its functionality to encapsulate some of the cases in which clients were attempting to use User Enrolment Rules.

  • Added the ability to specify an absolute end date for a User Enrolment Rule. User Enrolment Rules will stop processing when the end date is reached.
  • Fixed a bug where Subdomain specific message templates weren’t being used correctly by the enrolment rule resulting in the wrong email message being sent to learners enrolled via User Enrolment Rules. User Enrolment Rules now support subdomain specific message templates for enrolment.
  • Improved data logging for each User Enrolment Rule. All User Enrolment Rule run inputs are now visible via the Recent Activities section.
  • Improved performance of the scheduler’s handling of User Enrolment Rules so they are less likely to queue up or cause performance issue on the site.

Minor Changes

New Suite of Enrolment APIs

New APIs are available for creating, updating, and synchronising enrolments with 3rd party systems. These REST services use the same authentication mechanism as Report APIs.

New Suite of User APIs

New APIs for creating and updating user accounts with 3rd party systems. These REST services use the same authentication mechanism as Report APIs.

Other Learning 2-Stage Approval Workflow

Added 2-stage Approval Workflow to Other Learning, in addition to the existing 1-stage Approval Workflow.

Reverse SAML SSO

CLS tenants can now be configured to be SAML 2.0 identity providers, allowing you to use your CLS login details to login to other sites.
